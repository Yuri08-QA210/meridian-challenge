MERIDIAN

Post-Quantum Key Encapsulation Research Platform

SCHEME PARAMETERS

Dimension (n) 40 Modulus (q) 12289 (prime) Error bound {-1, 0, 1} Secret type Binary vector, Hamming weight = 20 Public key (A, b) where b = A*s + e mod q

The Key Encapsulation Mechanism (KEM) allows secure message transmission. A message bit m is encapsulated as ciphertext (u, v) where: u = A^T * r + e1 mod q, v = b^T * r + e2 + m * floor(q/2) mod q. Decapsulation recovers m from (u, v) using the private key s.

API ENDPOINTS

GET /api/info
Challenge parameters and configuration
GET /api/public_key
The public key (A, b) for encapsulation
GET /api/flag
The encapsulated flag ciphertexts (array of {u, v} pairs)
POST /api/decapsulate
Submit a ciphertext for decapsulation
  • Body: {"u": [int*40], "v": int}
  • Returns: {"bit": 0|1, "processing_time_us": float, "queries_remaining": int}
  • Use X-Session-ID header for session tracking
POST /api/session
Create a new session with fresh query counter

SERVICE NOTICE

This service implements a custom KEM for research purposes. Each session is limited to 10,000 decapsulation queries. The server reports its internal processing time for performance monitoring.

SAMPLE INTERACTION

$ curl /api/info
